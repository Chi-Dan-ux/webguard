WebGuard Documentation

Everything you need to know to scan, analyse, and report vulnerabilities in Caleb University's web applications.

Overview

WebGuard is a web-based automated vulnerability scanning and reporting system developed as part of a research project at Caleb University. It is designed specifically for university web environments — student portals, learning management systems (LMS), and administrative dashboards.

The system detects common web application vulnerabilities aligned with the OWASP Top 10 standards and generates multi-level reports: technical detail reports for IT personnel and executive summaries for non-technical administrators.

Automated scanning
Scans web apps for SQLi, XSS, auth weaknesses, and misconfigurations automatically.
Two-level reports
Technical reports for IT staff and plain-language executive summaries for administrators.
OWASP aligned
All detections map to the OWASP Top 10 — the globally recognised web security standard.

Getting Started

Follow these steps to run your first vulnerability scan with WebGuard.

1
Create your account

Click Get Started on the homepage. Register using your university email address and select your role — IT Personnel or Administrator.

2
Add a target

From the dashboard, click New Scan in the sidebar. Enter the full URL of the web application you want to assess, e.g. https://student-portal.caleb.edu.ng.

3
Configure scan modules

Select which vulnerability checks to run: SQL Injection, XSS, Authentication Weaknesses, Security Headers, and SSL/TLS. All are enabled by default for comprehensive coverage.

4
Review your report

When the scan completes, navigate to Reports to view and download your full vulnerability report as a PDF. IT personnel see technical details; administrators see the executive summary.

How It Works

WebGuard follows a four-stage pipeline for every scan:

1. Crawl Discover all pages, forms, and endpoints
2. Probe Inject test payloads into each input
3. Analyse Classify findings by severity and OWASP category
4. Report Generate multi-level PDF reports
Note: WebGuard performs non-destructive scanning. It tests for vulnerabilities without modifying or deleting any data on the target application.

Each scan module targets a specific class of vulnerability. You can enable or disable individual modules before starting a scan.

Understanding Reports

WebGuard generates two versions of every scan report, designed for different audiences:

Technical Report (IT Personnel)
  • Full vulnerability list with affected URL/endpoint
  • OWASP category and CVE references where applicable
  • Evidence — the exact request/response that triggered the finding
  • Severity score (Critical / High / Medium / Low)
  • Step-by-step remediation instructions
  • Code-level fix recommendations
Executive Summary (Administrators)
  • Overall security score (0–100)
  • Risk summary in plain language
  • Number of findings by severity level
  • Priority recommendations (no technical jargon)
  • Comparison with previous scan results
  • Downloadable one-page PDF

Frequently Asked Questions

Can WebGuard scan any website, or only Caleb University?
WebGuard can scan any publicly accessible web application. While it was built with Caleb University as the primary case study, you can scan student portals, LMS platforms, or admin dashboards at any institution by entering their URL.
Does scanning harm or modify the target website?
No. WebGuard performs passive and semi-passive scanning only. It sends crafted HTTP requests to detect vulnerabilities but never writes, deletes, or modifies any data on the target application. Think of it like a security auditor testing a lock — not breaking in.
How long does a scan take?
Scan duration depends on the size and complexity of the target application. A typical student portal takes 2–5 minutes. A large LMS with many pages and forms may take 10–15 minutes. You will see live progress in the dashboard while scanning.
What is the security score?
The security score is a 0–100 number calculated based on the number and severity of vulnerabilities found. A score of 80+ is considered Good. 60–79 is Fair — some issues need attention. Below 60 is Poor and requires urgent remediation.
Is this the same as OWASP ZAP?
WebGuard is a purpose-built system inspired by OWASP ZAP but designed specifically for university environments. The research validates WebGuard's detection accuracy through comparative testing against OWASP ZAP on intentionally vulnerable applications.
