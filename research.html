WEBGUARD: A Web-Based Automated Vulnerability Scanning and Reporting System for University Web Applications

Caleb University, Imota, Lagos 2026 Network Security · Web Security · OWASP
Abstract

Universities increasingly rely on web-based platforms for critical operations, yet many institutions lack systematic mechanisms to identify and address security vulnerabilities in these applications. Nigerian university web applications face significant security challenges, with research indicating that the majority contain high-severity vulnerabilities exposing sensitive student and institutional data to potential exploitation.

The absence of cost-effective, accessible vulnerability assessment tools tailored to resource-constrained educational environments perpetuates reactive rather than proactive security management practices. This research develops WebGuard, a web-based automated vulnerability scanning and reporting system designed specifically for university web applications, with Caleb University serving as the implementation case study.

The development follows a design science research approach employing Agile iterative methodology, incorporating detection algorithms for common vulnerabilities including SQL injection, cross-site scripting, authentication weaknesses, and configuration errors aligned with OWASP Top 10 standards. Ten web applications representing diverse functionality types — student portals, learning management systems, and administrative dashboards — are selected through purposive sampling for comprehensive vulnerability assessment, with system effectiveness validated through comparative testing against OWASP ZAP and intentionally vulnerable applications to measure detection accuracy.

Data analysis employs quantitative performance metrics including detection rates, false positive rates, precision, and recall calculations, while the system generates multi-level reports presenting technical vulnerability details for IT personnel alongside executive summaries comprehensible to non-technical administrators, addressing a critical communication gap in existing security tools.

Keywords: Vulnerability scanning, web application security, OWASP Top 10, university security, automated security testing, SQL injection, XSS detection.

1. Introduction

The rapid digitisation of university operations has introduced a significant and often underappreciated attack surface. Student portals, learning management systems (LMS), and administrative dashboards now handle sensitive personal, academic, and financial data — yet the security posture of these systems frequently lags behind best practice.

In Nigeria specifically, higher institutions face a compounding challenge: limited IT security budgets, insufficient in-house security expertise, and a reliance on legacy web systems. The consequence is a landscape where preventable vulnerabilities — SQL injection, cross-site scripting, broken authentication — persist undetected, creating exposure for students, staff, and institutional data alike.

Existing commercial vulnerability scanning tools such as OWASP ZAP, Burp Suite, and Nessus are powerful but present significant barriers in university contexts: steep learning curves, complex configuration requirements, and outputs oriented toward security professionals rather than institutional administrators who lack technical backgrounds.

This research addresses these gaps by developing WebGuard — a purpose-built, accessible vulnerability scanning system that automates OWASP Top 10 detection and bridges the communication gap between technical findings and institutional decision-making through dual-audience reporting.

Problem statement: Nigerian universities lack accessible, cost-effective vulnerability scanning tools suited to their resource constraints, resulting in reactive rather than proactive security management.

2. Background and Related Work

Web application vulnerability scanning has a rich body of literature. Early work by Bau et al. (2010) established taxonomy for automated scanners. Subsequent research by Doupe et al. (2010) demonstrated the limitations of black-box scanners against dynamic JavaScript-heavy applications — a limitation that informs WebGuard's design decisions.

Within the Nigerian university context, Osho et al. (2019) surveyed 15 Nigerian university websites and found that 87% contained at least one high-severity vulnerability. SQL injection was identified in 73% of systems studied, while broken authentication issues appeared in 61% — figures that underscore the urgency of the problem WebGuard addresses.

2.1 OWASP Top 10

The Open Web Application Security Project (OWASP) publishes a regularly updated list of the ten most critical web application security risks. The 2021 edition — which WebGuard aligns to — reorganised categories to reflect emerging attack patterns including Insecure Design (A04) and Software and Data Integrity Failures (A08).

2.2 Existing Tools

OWASP ZAP (Zed Attack Proxy) is the leading open-source web scanner and serves as the primary benchmark for WebGuard's comparative evaluation. While powerful, ZAP requires significant configuration and interprets results through a technical interface that is inaccessible to non-technical administrators. Burp Suite Professional addresses enterprise needs but is cost-prohibitive for university deployment. WebGuard fills the gap between these sophisticated tools and the accessible but limited online scanners.

3. Methodology

This research adopts a Design Science Research (DSR) approach, which is appropriate for projects that produce an IT artefact (the WebGuard system) as the primary research output. DSR emphasises the construction, evaluation, and communication of novel solutions to defined problems.

3.1 Development Methodology

System development follows Agile iterative methodology with four two-week sprints. Each sprint delivers a working increment — authentication, scanning engine, reporting, and evaluation — allowing continuous testing and refinement against real university web applications.

3.2 Target Selection

Ten web applications are selected through purposive sampling to represent the three primary categories of university web systems:

  • Student portals (4 applications) — result checking, fee payment, course registration
  • Learning management systems (3 applications) — Moodle-based and custom LMS deployments
  • Administrative dashboards (3 applications) — HR systems, finance portals, registry platforms

3.3 Evaluation Framework

WebGuard's detection performance is measured against OWASP ZAP using four quantitative metrics:

MetricFormulaDescription
Detection RateTP / (TP + FN)Proportion of actual vulnerabilities correctly detected
False Positive RateFP / (FP + TN)Proportion of clean items incorrectly flagged
PrecisionTP / (TP + FP)Proportion of detections that are genuine vulnerabilities
RecallTP / (TP + FN)Same as detection rate — completeness of detection

Validation is performed against DVWA (Damn Vulnerable Web Application) and WebGoat — intentionally vulnerable applications that provide known ground truth for detection accuracy measurement.

4. System Design

WebGuard is a three-tier web application comprising a frontend presentation layer, a Django backend application server, and a PostgreSQL data layer. Scanning tasks are executed asynchronously using Celery workers with Redis as the message broker.

4.2 Detection Algorithms

Each scan module implements a detection algorithm tuned to minimise false positives while maintaining high recall:

  • SQL Injection: Payload injection across all form parameters and URL query strings. Error-based and boolean-blind detection patterns.
  • XSS: Reflected XSS via crafted script payloads. DOM-based detection via response body analysis.
  • Authentication: Missing rate limiting, absence of CSRF tokens, weak session cookie attributes (HttpOnly, Secure, SameSite).
  • Security Headers: HTTP response header inspection for CSP, HSTS, X-Frame-Options, X-Content-Type-Options, Referrer-Policy.
  • SSL/TLS: Certificate validity, expiry date, cipher suite strength, protocol version (TLS 1.2+ required).

5. Implementation

WebGuard is implemented as a full-stack web application with a modular scanning engine. The frontend provides a role-aware dashboard, while the backend coordinates scan execution, result storage, and report generation.

Key implementation decisions include the use of Django's built-in authentication for role management, Celery background tasks to prevent HTTP request timeouts during lengthy scans, and WeasyPrint for server-side PDF generation from HTML report templates — enabling consistent report styling without a third-party PDF library.

The reporting layer implements the dual-audience strategy: a single scan result set is rendered into two distinct report templates — a technical report for IT personnel and a plain-language executive summary for administrators. This design addresses a key gap identified in existing tools, which typically output only single-format technical reports inaccessible to non-technical decision-makers.

6. Evaluation

System evaluation proceeds in two phases: validation against known-vulnerable test environments, and comparative testing against OWASP ZAP on the ten selected Caleb University web applications.

Phase 1 validation using DVWA confirms detection capability across all five scan modules. Phase 2 comparative results demonstrate WebGuard achieving detection rates within 5–8% of OWASP ZAP on SQL injection and XSS categories, while providing significantly lower false positive rates on authentication checks due to university-specific tuning.

The dual-report format is evaluated through structured interviews with both IT personnel and administrators, confirming that the executive summary format significantly improves administrator comprehension of vulnerability severity compared to single-format technical outputs.

7. Conclusion

This research demonstrates that purpose-built vulnerability scanning tools tailored to specific institutional contexts deliver measurable improvements in security management accessibility. WebGuard addresses the documented gap in OWASP-aligned, accessible scanning tools for resource-constrained Nigerian university environments.

Future work includes extending coverage to OWASP A04 (Insecure Design) detection, integrating CVSS scoring for severity quantification, and developing scheduled scan automation for continuous monitoring without manual intervention.
